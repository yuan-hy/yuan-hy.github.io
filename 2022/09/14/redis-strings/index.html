Redis strings

思路：通过记录 strings 的各个命令的含义以及用例来了解各个命令的用法

strings 包含以下命令

  • 通过 help @string 查看 string 字符串包含的所有命令的语法与命令描述，可以看出根据命令的字母顺序排序输出
$ redis-cli help @string 

  APPEND key value
  summary: Append a value to a key
  since: 2.0.0

  DECR key
  summary: Decrement the integer value of a key by one
  since: 1.0.0
  
  ...

  SUBSTR key start end
  summary: Get a substring of the string stored at a key
  since: 1.0.0
  • 因此根据以上可以列出字符串的所有命令
    • APPEND

预备知识

因为在记录 string 的命令时可能会用到以下还没介绍的命令，所以在这里预先记录一下

EXISTS

  • 语法：EXISTS key [key …] ,支持对多个 key 检查
  • 时间复杂度：O(N), N 表示要检查的 key 的数量，例如
    $ EXISTS key1,key2,key3,key4,key5,key6
  
这个时间复杂度为 O(6) 

    # 正常情况，当判断某一个 key 是否存在时，会根据具体情况返回整数，这个整数表示实际存在的 key 的数量
# key 不存在时返回 0
127.0.0.1:6379> exists key
(integer) 0
  
127.0.0.1:6379> set key "hello"
OK
# key 存在时，返回 1
127.0.0.1:6379> exists key
(integer) 1
  
127.0.0.1:6379> set key2 "world"
OK
# 当 key 为不同的并且存在的键时，返回的是实际存在的键数量
127.0.0.1:6379> exists key key2
(integer) 2
  
# 如果 key 存在且相同，也作为计算键的数量,两个相同的存在的 key ,返回值为 2
127.0.0.1:6379> exists key key
(integer) 2
  
# 当 key3 不存在时，是不作为计算返回的，这里 key3 不存在，key key2 存在，所以只返回 2
127.0.0.1:6379> exists key key2 key3
(integer) 2

TTL

  • 语法：TTL key

  • 时间复杂度：O(1)

  • 描述：获取某个 key 存活剩余的时间，单位为秒

    # 在 Redis 2.6 或更早版本， ttl key , 如果 key 不存在或者 key 存在，但是没有设置关联过期时间，返回 -1
  
# 而从 Redis 2.8 开始
# key 不存在，返回 -2
# key 存在，但是没有设置关联过期时间，返回 -1
  
# 与 pttl 命令类似，只不过 pttl 命令返回值单位为毫秒
  
  
127.0.0.1:6379> set expirekey "hello" # 创建 expirekey 的键
OK
127.0.0.1:6379> ttl expirekey
(integer) -1 # expirekey 存在，但是不关联过期时间，使用 ttl 命令返回 -1
127.0.0.1:6379> expire expirekey 10 # 给 expirekey 键关联存活时间
(integer) 1
127.0.0.1:6379> ttl expirekey # 每次执行返回的都是剩余的存活时间
(integer) 8
127.0.0.1:6379> ttl expirekey
(integer) 6
127.0.0.1:6379> ttl expirekey
(integer) 1
127.0.0.1:6379> ttl expirekey # 当 key 存活时间过期，expirekey 自动被删除，expirekey 不存在则返回 -2
(integer) -2
127.0.0.1:6379> get expirekey # 当 key 的存活时间超时再去获取时， key 已经不存在了，返回 nil
(nil)

每个命令的用法

APPEND

  • 语法：APPEND key value
  • 描述：给某个 key 追加 value 值
  • 时间复杂度：O(1)
  • 参考官方 append 命令
    # 当 key 不存在时，使用 append 命令给某个 key 追加值时，会先创建该 key ,赋予空字符串，然后相当于 set 命令，给创建的 key 设值
127.0.0.1:6379> exists mykey
(integer) 0
127.0.0.1:6379> append mykey "value"
(integer) 5 # append 命令的返回值表示追加操作后的字符串长度。
127.0.0.1:6379> get mykey
"value"
  
# 当 mykey 这个键存在值后，append 时，会将值追加到当前值的后面
127.0.0.1:6379> append mykey "hello"
(integer) 10 # append 命令的返回值表示追加操作后的字符串长度。上面 value 的长度为 5，追加长度为 5 的 hello 后，返回 10
127.0.0.1:6379> get mykey
"valuehello"
  
# 可以作为时间序列使用，大概意思就是每次追加的这个值的长度是固定长度的
# 语法：APPEND timeseries "fixed-size sample"
# 访问时间序列中的单个元素并不难：
  # STRLEN可用于获取样本数。
  # GETRANGE允许随机访问元素。GETRANGE如果我们的时间序列有相关的时间信息，我们可以很容易地结合Redis 2.6 中可用的 Lua 脚本引擎实现二进制搜索来获取范围。
  # SETRANGE可用于覆盖现有的时间序列。
   
# 使用固定大小的字符串对传感器温度进行采样的示例（在实际实现中使用二进制格式更好）
127.0.0.1:6379> APPEND ts "0043"
(integer) 4
127.0.0.1:6379> APPEND ts "0035"
(integer) 8
127.0.0.1:6379> GETRANGE ts 0 3
"0043"
127.0.0.1:6379> GETRANGE ts 4 7
"0035"

DECR

  • 语法：DECR key

  • 描述：某个 key 对应的 value 值为整数，DECR key 表示给存储在某个 key 的整数值减 1

  • 时间复杂度：O(1)

  • 参考官方 decr 命令

    # decr 命令返回值为减 1 之后的整数值，如 下面的返回值 9 = 10 - 1
127.0.0.1:6379> SET mykey "10"
"OK"
127.0.0.1:6379> DECR mykey
(integer) 9
# 当 decr 操作的 key 不存在时，在操作减 1 之前，会将这个 key 设置为 0 ，再执行减 1 操作
127.0.0.1:6379> exists dec
(integer) 0
127.0.0.1:6379> decr dec
(integer) -1
127.0.0.1:6379> decr dec
(integer) -2
127.0.0.1:6379> exists dec # 此时 dec 这个 key 在设置为 0 时已被创建
(integer) 1
  
# key 对应的 value 值范围是：64 位的有符号整数，即 -2^63~2^63-1
127.0.0.1:6379> SET mykey "234293482390480948029348230948"
(integer) 0
127.0.0.1:6379> decr mykey
(error) ERR value is not an integer or out of range # 超过了限制长度范围，报错了
# 当 key 对应的值不为整数或者超出限制范围，都会报如下错误
127.0.0.1:6379> get key
"hello"
127.0.0.1:6379> decr key
(error) ERR value is not an integer or out of range
  
127.0.0.1:6379> set flag true
OK
127.0.0.1:6379> decr flag
(error) ERR value is not an integer or out of range

DECRBY

  • 语法：DECRBY key decrement

  • 描述：某个 key 对应的 value 值为整数，DECRBY key decrement 表示给存储在某个 key 的整数值减去给定的 decrement ，返回操作后的值

  • 时间复杂度：O(1)

  • 参考官方 decrby 命令

    # decrby 命令返回值为减去指定数字值 之后的整数值，如下面的返回值 6 = 10 - 4
127.0.0.1:6379> set mykey 10
OK
127.0.0.1:6379> decrby mykey 4
(integer) 6
# 当 decrby 操作的 key 不存在时，在操作减 之前，会将这个 key 设置为 0 ，再执行减 4 操作
127.0.0.1:6379> getdel dec
"-2"
127.0.0.1:6379> exists dec
(integer) 0
127.0.0.1:6379> decrby dec 4
(integer) -4
127.0.0.1:6379> exists dec # 此时 dec 这个 key 在设置为 0 时已被创建
(integer) 1
  
# key 对应的 value 值范围是：64 位的有符号整数，即 -2^63~2^63-1
127.0.0.1:6379> SET mykey "234293482390480948029348230948"
(integer) 0
127.0.0.1:6379> decrby mykey 4
(error) ERR value is not an integer or out of range # 超过了限制长度范围，报错了
# 当 key 对应的值不为整数或者超出限制范围，都会报如下错误
127.0.0.1:6379> get key
"hello"
127.0.0.1:6379> decrby key 4
(error) ERR value is not an integer or out of range
  
127.0.0.1:6379> set flag true
OK
127.0.0.1:6379> decrby flag 4
(error) ERR value is not an integer or out of range

GET

  • 语法：GET key

  • 描述：获取某个 key 的 value 值

  • 时间复杂度：O(1)

  • 参考官方 get 命令

    # get 命令返回 key 的字符串值
127.0.0.1:6379> get key
"hello"
  
# 当 key 不存在时，返回 nil
127.0.0.1:6379> get nonexisting
(nil)
  
# get 只处理 string 的值

GETDEL

  • 语法：GETDEL key

  • 描述：获取返回 key 的值并删除这个 key,与 get 用法类似，只是返回值时同时删除 key

  • 时间复杂度：O(1)

  • 参考官方 getdel 命令

    127.0.0.1:6379> set mykey "hello"
OK
# 获取返回 key 的值并删除 mykey
127.0.0.1:6379> getdel mykey
"hello"
127.0.0.1:6379> getdel mykey
(nil)
127.0.0.1:6379> get mykey
(nil)
  
# 当 key 不存在时，返回 nil
127.0.0.1:6379> getdel nonexisting
(nil)
  
# getdel 只处理 string 的值

GETEX

  • 语法：GETEX key [EX seconds|PX milliseconds|EXAT unix-time-seconds|PXAT unix-time-milliseconds|PERSIST]

  • 描述：获取返回 key 的值,如果使用选项，则可以给这个 key 设置过期时间

  • 时间复杂度：O(1)

  • 参考官方 getex 命令

  • 额外选项

    • EX seconds ：设置指定的过期时间，以秒为单位
    • PX milliseconds ：设置指定的过期时间，以毫秒为单位
    • EXAT unix-time-seconds ：设置密钥过期的指定 Unix 时间，以秒为单位
    • PXAT unix-time-milliseconds ：设置密钥过期的指定 Unix 时间，以毫秒为单位。
    • PERSIST ：删除与密钥关联的生存时间。
    127.0.0.1:6379> set expirekey "hello"
OK
# 仅仅只是获取时间，并没有设置关联过期时间
127.0.0.1:6379> getex expirekey
"hello"
127.0.0.1:6379> ttl expirekey
(integer) -1
# 获取时间，并设置关联过期时间
127.0.0.1:6379> getex expirekey ex 10
"hello"
127.0.0.1:6379> ttl expirekey
(integer) 1
127.0.0.1:6379> ttl expirekey
(integer) -2
  
# 通过 persist 选项删除关联的过期时间
127.0.0.1:6379> getex expirekey ex 100
"hello"
127.0.0.1:6379> ttl expirekey
(integer) 93
127.0.0.1:6379> getex expirekey persist
"hello"
127.0.0.1:6379> ttl expirekey
(integer) -1

GETRANGE

  • 语法：GETRANGE key start end
  • 描述：获取存储在一个键中的字符串的子串
  • 时间复杂度：O(N)，N 表示截取返回字符串的长度，但小字符串的复杂度认为是 O(1)
  • 参考官方 getrange 命令
  • 额外选项
    • start ：截取起始位置(包含在内)
    • end ：截取结束位置(包含在内)
    • 支持负数偏移量，例如 -1 表示字符串最后一个字符的位置，-2 表示倒数第二位，-3 即倒数第三位
    127.0.0.1:6379> SET mykey "This is a string"
"OK"
127.0.0.1:6379> GETRANGE mykey 0 3
"This"
127.0.0.1:6379> GETRANGE mykey -3 -1 # 截取从倒数第三位到倒数第一位
"ing"
127.0.0.1:6379> GETRANGE mykey 0 -1
"This is a string"
127.0.0.1:6379> GETRANGE mykey 10 100
"string"

GETSET

  • Redis 6.2.0 版本废弃了，使用 set 与 get 操作替代
  • 语法：GETSET key value
  • 描述：给 key 设置新值，返回旧值

INCR

  • 语法：INCR key

  • 描述：某个 key 对应的 value 值为整数，INCR key 表示给存储在某个 key 的整数值加 1

  • 时间复杂度：O(1)

  • 参考官方 incr 命令

    # incr 命令返回值为加 1 之后的整数值，如 下面的返回值 11 = 10 + 1
127.0.0.1:6379> SET mykey "10"
"OK"
127.0.0.1:6379> incr mykey
(integer) 11
# 当 incr 操作的 key 不存在时，在操作加 1 之前，会将这个 key 设置为 0 ，再执行加 1 操作
127.0.0.1:6379> exists inc
(integer) 0
127.0.0.1:6379> incr inc
(integer) 1
127.0.0.1:6379> incr inc
(integer) 2
127.0.0.1:6379> exists inc # 此时 inc 这个 key 在设置为 0 时已被创建
(integer) 1
  
# key 对应的 value 值范围是：64 位的有符号整数，即 -2^63~2^63-1
127.0.0.1:6379> SET mykey "234293482390480948029348230948"
(integer) 0
127.0.0.1:6379> incr mykey
(error) ERR value is not an integer or out of range # 超过了限制长度范围，报错了
# 当 key 对应的值不为整数或者超出限制范围，都会报如下错误
127.0.0.1:6379> get key
"hello"
127.0.0.1:6379> incr key
(error) ERR value is not an integer or out of range
  
127.0.0.1:6379> set flag true
OK
127.0.0.1:6379> incr flag
(error) ERR value is not an integer or out of range

INCRBY

  • 语法：INCRBY key increment

  • 描述：某个 key 对应的 value 值为整数，INCRBY key decrement 表示给存储在某个 key 的整数值减去给定的 decrement ，返回操作后的值

  • 时间复杂度：O(1)

  • 参考官方 incrby 命令

    # incrby 命令返回值为加上指定数字值 increment 之后的整数值，如下面的返回值 14 = 10 + 4
127.0.0.1:6379> set mykey 10
OK
127.0.0.1:6379> incrby mykey 4
(integer) 14
# 当 incrby 操作的 key 不存在时，在操作加之前，会将这个 key 设置为 0 ，再执行加 4 操作
127.0.0.1:6379> getdel inc
"-2"
127.0.0.1:6379> exists inc
(integer) 0
127.0.0.1:6379> incrby inc 4
(integer) 4
127.0.0.1:6379> exists inc # 此时 inc 这个 key 在设置为 0 时已被创建
(integer) 1
  
# key 对应的 value 值范围是：64 位的有符号整数，即 -2^63~2^63-1
127.0.0.1:6379> SET mykey "234293482390480948029348230948"
(integer) 0
127.0.0.1:6379> incrby mykey 4
(error) ERR value is not an integer or out of range # 超过了限制长度范围，报错了
# 当 key 对应的值不为整数或者超出限制范围，都会报如下错误
127.0.0.1:6379> get key
"hello"
127.0.0.1:6379> incrby key 4
(error) ERR value is not an integer or out of range
  
127.0.0.1:6379> set flag true
OK
127.0.0.1:6379> incrby flag 4
(error) ERR value is not an integer or out of range

INCRBYFLOAT

  • 语法：INCRBYFLOAT key increment

  • 描述：INCRBYFLOAT key increment 表示给存储在某个 key 的数值(支持浮点数)加 increment(支持浮点数)，返回增加 increment 后的值

  • 时间复杂度：O(1)

  • 参考官方 incrbyfloat 命令

    127.0.0.1:6379> SET mykey "10"
OK
127.0.0.1:6379> incrbyfloat mykey 1 # 可以增加正整数
"11"
127.0.0.1:6379> incrbyfloat mykey 0.5 # 可以加浮点数(小数)
"11.5"
127.0.0.1:6379> incrbyfloat mykey -1 # 可以加负数，就是减法
"10.5"
127.0.0.1:6379> exists noexistskey
(integer) 0
127.0.0.1:6379> incrbyfloat noexistskey -1 # 不存在时先设置为 0 ，后再执行操作，-1 表示减 1
"-1"
127.0.0.1:6379> set mykey 5.0e3 # 支持为指数
OK
127.0.0.1:6379> incrbyfloat mykey 2.0e2 # 支持为指数
"5200"

LCS

  • 语法：LCS key1 key2 [LEN] [IDX] [MINMATCHLEN len] [WITHMATCHLEN]

  • 描述：找出两个字符串最长的公共子串

  • 时间复杂度：O(N*M) N 和 M 分别是 key1 和 key2 字符串的长度

  • 参考官方 incrbyfloat 命令

  • 大意就是从左到右匹配子序列，将选出公共的子序列，可以不连续

  • 参数

    • len 长度，匹配输出最长的公共子序列
    # 不加参数选项匹配
127.0.0.1:6379> MSET key1 ohmytext key2 mynewtext
OK
127.0.0.1:6379> lcs key1 key2
"mytext"

# 127.0.0.1:6379> lcs key1 key2 len
(integer) 6

127.0.0.1:6379> lcs key1 key2 idx
1) "matches"
2) 1) 1) 1) (integer) 4
         2) (integer) 7
      2) 1) (integer) 5
         2) (integer) 8
   2) 1) 1) (integer) 2
         2) (integer) 3
      2) 1) (integer) 0
         2) (integer) 1
3) "len"
4) (integer) 6

MGET

  • 语法：MGET key [key ...]
  • 描述：获取所有给定的 key 的值，支持多个 key ,key 之间空格隔开
  • 时间复杂度：O(N) ，其中 N 是要检索的键的数量
  • 参考官方 mget 命令
    # 返回所有指定检索 key 的值，如果该 key 不存在或者不持有 string 值，则返回 nil ,这个操作永远不会失败 
127.0.0.1:6379> set key1 "hello"
OK
127.0.0.1:6379> set key2 "world"
OK
127.0.0.1:6379> set key3 ""
OK
127.0.0.1:6379> mget key1 key2 key3 noexistkey
1) "hello"
2) "world"
3) ""
4) (nil)

MSET

  • 语法：MSET key value [key value ...]

  • 描述：批量给指定的 key 设值，格式为 key value 为一组

  • 时间复杂度：O(N) ，其中 N 是要设置的 key 的数量

  • 参考官方 mset 命令

    # MSET 命令会使用新值覆盖旧值，原子操作，与 set 命令效果一样。如果不想覆盖旧值，可以考虑 MSETNX  
127.0.0.1:6379> set key1 "hello"
OK
127.0.0.1:6379> set key2 "world"
OK
127.0.0.1:6379> set key3 ""
OK
127.0.0.1:6379> mset key1 "hellomset" key2 "worldset"
OK
127.0.0.1:6379> mget key1 key2 key3 noexistkey
1) "hellomset"
2) "worldset"
3) ""
4) (nil)
# key1 ,key2 存在值，使用 msetnx 命令设置值时，返回 0
127.0.0.1:6379> msetnx key1 "hello" key2 "world"
(integer) 0
127.0.0.1:6379> mget key1 key2 key3 noexistkey # 获取 key1,key2 的值不变，表示没有设值成功
1) "hellomset"
2) "worldset"
3) ""
4) (nil)
# key4 不存在，使用 msetnx 命令设置值时，返回 1
127.0.0.1:6379> msetnx key4 "msetnx"
(integer) 1
127.0.0.1:6379> mget key1 key2 key3 noexistkey key4 # 获取 key4 ，设值成功
1) "hellomset"
2) "worldset"
3) ""
4) (nil)
5) "msetnx"

MSETNX

  • 语法：MSETNX key value [key value ...]

  • 描述：批量给指定的 key 设值，格式为 key value 为一组，只有 key 不存在才成功,即使只有其中一个 key 存在值，所有的都不会设置成功

  • 时间复杂度：O(N) ，其中 N 是要设置的 key 的数量

  • 参考官方 msetnx 命令

    # MSETNX 命令不会使用新值覆盖旧值  
# key1 ,key2 存在值，使用 msetnx 命令设置值时，即使有一个 key 有值，都是不会设置成功的，返回 0
127.0.0.1:6379> msetnx key1 "hello" key2 "world"
(integer) 0
127.0.0.1:6379> mget key1 key2 key3 noexistkey # 获取 key1,key2 的值不变，表示没有设值成功
1) "hellomset"
2) "worldset"
3) ""
4) (nil)
# key4 不存在，使用 msetnx 命令设置值时，所有 key 设置成功返回 1
127.0.0.1:6379> msetnx key4 "msetnx"
(integer) 1
127.0.0.1:6379> mget key1 key2 key3 noexistkey key4 # 获取 key4 ，设值成功
1) "hellomset"
2) "worldset"
3) ""
4) (nil)
5) "msetnx"

    PSETEX

  • 语法：PSETEX key milliseconds value

  • 描述：给某个 key 设置过期时间和值，过期时间的单位为 毫秒

  • 时间复杂度：O(1)

  • 参考官方 psetex 命令

    # 工作原理与 setex 唯一区别就是 psetex 的过期时间单位是毫秒而不是秒
# key1 ,key2 存在值，使用 msetnx 命令设置值时，即使有一个 key 有值，都是不会设置成功的，返回 0
127.0.0.1:6379> PSETEX mykey 10000 "Hello"
"OK"
127.0.0.1:6379> PTTL mykey
(integer) 7682
127.0.0.1:6379> GET mykey
"Hello"
# 过期时间到了，mykey 不存在了，获取值为 nil ,通过 pttl 查看返回 -2
127.0.0.1:6379> get mykey
(nil)
127.0.0.1:6379> pttl mykey
(integer) -2

SET

  • 语法：SET key value [NX|XX] [GET] [EX seconds|PX milliseconds|EXAT unix-time-seconds|PXAT unix-time-milliseconds|KEEPTTL]

  • 描述：给某个 key 设置值，如果该 key 存在值，则覆盖，而且无视旧值的类型，任何先前与该键关联的生存时间在成功SET操作时都将被丢弃。

  • 时间复杂度：O(1)

  • 参考官方 set 命令

  • 参数选项

    • EX seconds – 设置指定的过期时间，以秒为单位。
    • PX 毫秒——设置指定的过期时间，以毫秒为单位。
    • EXAT timestamp-seconds – 设置密钥过期的指定 Unix 时间，以秒为单位。
    • PXAT timestamp-milliseconds – 设置密钥过期的指定 Unix 时间，以毫秒为单位。
    • NX– 仅当密钥不存在时才设置它。
    • XX– 仅当密钥已存在时才设置它。
    • KEEPTTL– 保留与密钥关联的生存时间。
    • GET– 返回存储在 key 中的旧字符串，如果 key 不存在，则返回 nil。SET如果存储在 key 的值不是字符串，则返回并中止错误。
    • 注意：由于SET命令选项可以替换SETNX, SETEX, PSETEX, GETSET，因此在未来的 Redis 版本中，这些命令可能会被弃用并最终被删除。
    127.0.0.1:6379> SET mykey "Hello"
"OK"
127.0.0.1:6379> GET mykey
"Hello"
127.0.0.1:6379> SET anotherkey "will expire in a minute" EX 60
"OK"
127.0.0.1:6379>

SETEX

  • 语法：SETEX key seconds value

  • 描述：给某个 key 设置值和过期时间，如果该 key 存在值，则覆盖，而且无视旧值的类型，任何先前与该键关联的生存时间在成功SET操作时都将被丢弃。

  • 时间复杂度：O(1)

  • 参考官方 setex 命令

    # setex 相当于以下命令
SET mykey value
EXPIRE mykey seconds

# SETEX 是原子的，可以通过在MULTI/EXEC块中使用前两个命令来重现。它是作为给定操作序列的更快替代方案提供的，因为当 Redis 用作缓存时，此操作非常常见。
127.0.0.1:6379> SETEX mykey 10 "Hello"
"OK"
127.0.0.1:6379> TTL mykey
(integer) 10
127.0.0.1:6379> GET mykey
"Hello"

SETNX

  • 语法：SETNX key value

  • 描述：仅当 key 不存在值时给某个 key 设置值，否则不执行设值

  • 时间复杂度：O(1)

  • 参考官方 setnx 命令

    # key 不存在时返回 1
127.0.0.1:6379> SETNX mykey "Hello"
(integer) 1
# key 存在值时，命令不执行，返回 0
127.0.0.1:6379> SETNX mykey "World"
(integer) 0
127.0.0.1:6379> GET mykey
"Hello"

SETRANGE

  • 语法：SETRANGE key offset value

  • 描述：覆盖存储在key的部分字符串，从指定的偏移量 offset 开始，覆盖 value 的整个长度

  • 时间复杂度：O(1)

  • 参考官方 setrange 命令

    127.0.0.1:6379> SET key1 "Hello World"
OK
127.0.0.1:6379> SETRANGE key1 6 "Redis"
(integer) 11 # 命令返回值为修改后的字符串长度
127.0.0.1:6379> GET key1
"Hello Redis"

# 不存在的密钥当作空值，key5 不存在，所以执行命令时创建了 key5 ,持有空值
127.0.0.1:6379> SETRANGE key5 6 "Redis"
(integer) 11
127.0.0.1:6379> get key5 # 当偏移量 6 大于原本 key5 的值的长度时，会在前面补 0
"\x00\x00\x00\x00\x00\x00Redis"

STRLEN

  • 语法：STRLEN key

  • 描述：返回这个存储在这个 key 的字符串值的长度，如果该值不是字符串，返回错误。 key 不存在返回 0

  • 时间复杂度：O(1)

  • 参考官方 strlen 命令

    redis> SET mykey "Hello world"
"OK"
redis> STRLEN mykey
(integer) 11
redis> STRLEN nonexisting
(integer) 0

SUBSTR

  • 语法：SUBSTR key start end
  • 描述：截取存储在 key 的子字符串，起始位置为 start,结束位置为 end
  • 时间复杂度：O(N)
  • redis 2.0.0 版本过期了，getrange 替换
  • 参考官方 substr 命令
